## Getting Started

This guide will walk you through creating a simple Flask application with Discord OAuth2 authentication.

<Steps>

<Step title="Set up your Discord application">

Before you begin, create a Discord application:

1. Go to the [Discord Developer Portal](https://discord.com/developers/applications)
2. Click "New Application" and give it a name
3. Navigate to the "OAuth2" section
4. Note your **Client ID** and **Client Secret**
5. Add a redirect URI (e.g., `http://localhost:5000/callback/`)

<Warning>
  Keep your Client Secret confidential. Never commit it to version control.
</Warning>

</Step>

<Step title="Create your Flask application">

Create a new Python file (e.g., `app.py`) and add the following code:

```python
import os

from flask import Flask, redirect, url_for
from flask_discord import DiscordOAuth2Session, requires_authorization, Unauthorized

app = Flask(__name__)

app.secret_key = b"random bytes representing flask secret key"
os.environ["OAUTHLIB_INSECURE_TRANSPORT"] = "true"      # !! Only in development environment.

app.config["DISCORD_CLIENT_ID"] = 490732332240863233    # Discord client ID.
app.config["DISCORD_CLIENT_SECRET"] = ""                # Discord client secret.
app.config["DISCORD_REDIRECT_URI"] = ""                 # URL to your callback endpoint.
app.config["DISCORD_BOT_TOKEN"] = ""                    # Required to access BOT resources.

discord = DiscordOAuth2Session(app)
```

<Note>
  Replace the placeholder values with your actual Discord application credentials.
</Note>

<Warning>
  The `OAUTHLIB_INSECURE_TRANSPORT` setting should **only** be used in development. Remove it in production and use HTTPS.
</Warning>

</Step>

<Step title="Create the login route">

Add a route that initiates the OAuth2 flow:

```python
@app.route("/login/")
def login():
    return discord.create_session()
```

The `create_session()` method redirects users to Discord for authorization.

</Step>

<Step title="Create the callback route">

Add a callback route to handle the OAuth2 response:

```python
@app.route("/callback/")
def callback():
    discord.callback()
    return redirect(url_for(".me"))
```

This route processes the authorization code and saves the token to the session.

</Step>

<Step title="Add unauthorized error handler">

Handle unauthorized access attempts:

```python
@app.errorhandler(Unauthorized)
def redirect_unauthorized(e):
    return redirect(url_for("login"))
```

This redirects users to the login page when they try to access protected routes without authorization.

</Step>

<Step title="Create a protected route">

Add a route that displays the authenticated user's information:

```python
@app.route("/me/")
@requires_authorization
def me():
    user = discord.fetch_user()
    return f"""
    <html>
        <head>
            <title>{user.name}</title>
        </head>
        <body>
            <img src='{user.avatar_url}' />
        </body>
    </html>"""
```

The `@requires_authorization` decorator ensures only authenticated users can access this route.

</Step>

<Step title="Run your application">

Add the following to run your Flask app:

```python
if __name__ == "__main__":
    app.run()
```

Run your application:

```bash
python app.py
```

Visit `http://localhost:5000/login/` in your browser to test the OAuth2 flow.

</Step>

</Steps>

## Complete Example

Here's the complete working example:

```python
import os

from flask import Flask, redirect, url_for
from flask_discord import DiscordOAuth2Session, requires_authorization, Unauthorized

app = Flask(__name__)

app.secret_key = b"random bytes representing flask secret key"
os.environ["OAUTHLIB_INSECURE_TRANSPORT"] = "true"      # !! Only in development environment.

app.config["DISCORD_CLIENT_ID"] = 490732332240863233    # Discord client ID.
app.config["DISCORD_CLIENT_SECRET"] = ""                # Discord client secret.
app.config["DISCORD_REDIRECT_URI"] = ""                 # URL to your callback endpoint.
app.config["DISCORD_BOT_TOKEN"] = ""                    # Required to access BOT resources.

discord = DiscordOAuth2Session(app)


@app.route("/login/")
def login():
    return discord.create_session()
	

@app.route("/callback/")
def callback():
    discord.callback()
    return redirect(url_for(".me"))


@app.errorhandler(Unauthorized)
def redirect_unauthorized(e):
    return redirect(url_for("login"))

	
@app.route("/me/")
@requires_authorization
def me():
    user = discord.fetch_user()
    return f"""
    <html>
        <head>
            <title>{user.name}</title>
        </head>
        <body>
            <img src='{user.avatar_url}' />
        </body>
    </html>"""


if __name__ == "__main__":
    app.run()
```

## Configuration Options

The following configuration options are available:

| Config Key | Description | Required |
|------------|-------------|----------|
| `DISCORD_CLIENT_ID` | Your Discord application's client ID | Yes |
| `DISCORD_CLIENT_SECRET` | Your Discord application's client secret | Yes |
| `DISCORD_REDIRECT_URI` | The callback URL for OAuth2 | Yes |
| `DISCORD_BOT_TOKEN` | Bot token for accessing bot resources | No |
| `DISCORD_USERS_CACHE_MAX_LIMIT` | Maximum number of users to cache | No |

<Tip>
  You can also pass these values directly to the `DiscordOAuth2Session` constructor instead of using Flask config.
</Tip>
