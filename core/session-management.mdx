Flask-Discord uses Flask sessions to store authorization tokens and maintain user authentication state. This page explains how session management works and how to customize it for your needs.

## Default Session Handling

By default, Flask-Discord stores authorization tokens in client-side Flask sessions (cookies). This provides a simple, stateless authentication mechanism.

### How It Works

<Steps>
  <Step title="User Authorization">
    When a user completes OAuth2 authorization, Discord returns an access token
  </Step>
  <Step title="Token Storage">
    The `callback()` method calls `save_authorization_token()` to store the token in the Flask session
  </Step>
  <Step title="Token Retrieval">
    When making API requests, `get_authorization_token()` retrieves the token from the session
  </Step>
  <Step title="Token Usage">
    The token is automatically included in requests to Discord's API
  </Step>
</Steps>

## Token Storage Methods

Flask-Discord provides two static methods for managing authorization tokens:

### save_authorization_token()

This method saves the OAuth2 token to the user's session:

```python
@staticmethod
def save_authorization_token(token: dict):
    """Saves Discord OAuth2 token to the user's session."""
    session["DISCORD_OAUTH2_TOKEN"] = token
```

The token dictionary contains:
- `access_token`: The OAuth2 access token
- `token_type`: Usually "Bearer"
- `expires_in`: Token expiration time in seconds
- `refresh_token`: Token for refreshing the access token
- `scope`: List of granted scopes

### get_authorization_token()

This method retrieves the stored token:

```python
@staticmethod
def get_authorization_token() -> dict:
    """Returns the Discord OAuth2 token from the user's session."""
    return session.get("DISCORD_OAUTH2_TOKEN")
```

<Info>
These methods are called automatically by Flask-Discord. You rarely need to call them directly.
</Info>

## Server-Side Session Storage

For production applications, you may want to store tokens server-side instead of in cookies. This is more secure and allows better control over session management.

### Custom Session Backend

Override both `save_authorization_token()` and `get_authorization_token()` to implement server-side storage:

```python
from flask_discord import DiscordOAuth2Session
from flask import session
import redis
import json

# Initialize Redis client
redis_client = redis.Redis(host='localhost', port=6379, db=0)

class CustomDiscordSession(DiscordOAuth2Session):
    @staticmethod
    def save_authorization_token(token: dict):
        """Save token to Redis with user session ID."""
        session_id = session.get("session_id")
        if session_id:
            redis_client.setex(
                f"discord_token:{session_id}",
                3600,  # 1 hour expiration
                json.dumps(token)
            )
    
    @staticmethod
    def get_authorization_token() -> dict:
        """Retrieve token from Redis."""
        session_id = session.get("session_id")
        if session_id:
            token_data = redis_client.get(f"discord_token:{session_id}")
            if token_data:
                return json.loads(token_data)
        return None

# Use the custom session class
discord = CustomDiscordSession(app)
```

### Database Storage Example

Store tokens in a database for persistent session management:

```python
from flask_discord import DiscordOAuth2Session
from flask import session
from your_app.models import UserToken, db

class DatabaseDiscordSession(DiscordOAuth2Session):
    @staticmethod
    def save_authorization_token(token: dict):
        """Save token to database."""
        user_id = session.get("user_id")
        if user_id:
            user_token = UserToken.query.filter_by(user_id=user_id).first()
            if user_token:
                user_token.access_token = token["access_token"]
                user_token.refresh_token = token["refresh_token"]
                user_token.expires_in = token["expires_in"]
            else:
                user_token = UserToken(
                    user_id=user_id,
                    access_token=token["access_token"],
                    refresh_token=token["refresh_token"],
                    expires_in=token["expires_in"]
                )
                db.session.add(user_token)
            db.session.commit()
    
    @staticmethod
    def get_authorization_token() -> dict:
        """Retrieve token from database."""
        user_id = session.get("user_id")
        if user_id:
            user_token = UserToken.query.filter_by(user_id=user_id).first()
            if user_token:
                return {
                    "access_token": user_token.access_token,
                    "refresh_token": user_token.refresh_token,
                    "expires_in": user_token.expires_in,
                    "token_type": "Bearer"
                }
        return None

discord = DatabaseDiscordSession(app)
```

<Warning>
When overriding session methods, you must override BOTH `save_authorization_token()` and `get_authorization_token()`. Overriding only one will cause errors.
</Warning>

## Session Keys

Flask-Discord stores multiple values in the Flask session:

| Session Key | Purpose |
|-------------|----------|
| `DISCORD_OAUTH2_TOKEN` | The OAuth2 access token and metadata |
| `DISCORD_OAUTH2_STATE` | CSRF protection state during OAuth flow |

These keys are managed automatically and should not be modified directly.

## Checking Authorization Status

Use the `authorized` property to check if a user has a valid token:

```python
@app.route("/dashboard")
def dashboard():
    if not discord.authorized:
        return redirect(url_for("login"))
    
    user = discord.fetch_user()
    return render_template("dashboard.html", user=user)
```

The `authorized` property checks if:
1. An authorization token exists in the session
2. The token has not expired

## Revoking Sessions

The `revoke()` method clears all Discord-related data from the session:

```python
@app.route("/logout")
def logout():
    discord.revoke()
    return redirect(url_for("index"))
```

This method:
1. Removes the user from the internal cache
2. Clears `DISCORD_OAUTH2_TOKEN` from the session
3. Clears `DISCORD_OAUTH2_STATE` from the session

<Note>
The `revoke()` method only clears local session data. It does not revoke the token with Discord. Users can still use the access token until it expires.
</Note>

## User Caching

Flask-Discord maintains an internal cache of authorized users to reduce API calls. This cache is separate from session storage.

### Default Cache

By default, Flask-Discord uses `cachetools.LFUCache` with a maximum of 100 users:

```python
discord = DiscordOAuth2Session(app)
# Uses LFUCache with max 100 users
```

### Custom Cache Configuration

Configure the cache size in your Flask config:

```python
app.config["DISCORD_USERS_CACHE_MAX_LIMIT"] = 500
```

### Custom Cache Implementation

Provide your own cache implementation:

```python
from cachetools import TTLCache

# Cache with 1-hour TTL
custom_cache = TTLCache(maxsize=200, ttl=3600)

discord = DiscordOAuth2Session(app, users_cache=custom_cache)
```

## Best Practices

<Accordion title="Use server-side sessions for production">
Client-side sessions (cookies) have size limitations and security considerations. For production applications, implement server-side session storage using Redis or a database.
</Accordion>

<Accordion title="Set appropriate token expiration">
Discord access tokens expire after a period of time. Implement token refresh logic or require users to re-authenticate when tokens expire.
</Accordion>

<Accordion title="Secure your SECRET_KEY">
The Flask `SECRET_KEY` is used to sign session cookies and JWT tokens. Use a strong, random key and keep it secret:

```python
import secrets
app.config["SECRET_KEY"] = secrets.token_hex(32)
```
</Accordion>

<Accordion title="Handle token expiration gracefully">
Check if the user is authorized before making API requests:

```python
@app.route("/api/user")
def api_user():
    if not discord.authorized:
        return {"error": "Not authorized"}, 401
    
    user = discord.fetch_user()
    return user.to_json()
```
</Accordion>
