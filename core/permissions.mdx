Discord's permission system controls what actions users and bots can perform in guilds and channels. Flask-Discord provides a `Permissions` class to easily work with Discord permissions when adding bots to servers.

## Overview

The `Permissions` class is used when adding bots to guilds through OAuth2 with the `bot` scope. It allows you to specify exactly what permissions your bot should have.

```python
from flask_discord.types import Permissions
from flask_discord import DiscordOAuth2Session

@app.route("/add-bot")
def add_bot():
    perms = Permissions(send_messages=True, manage_messages=True)
    return discord.create_session(scope=["bot"], permissions=perms)
```

## Creating Permissions

### Using Keyword Arguments

The most intuitive way to create permissions is using keyword arguments:

```python
perms = Permissions(
    send_messages=True,
    manage_messages=True,
    embed_links=True,
    attach_files=True
)
```

### Using Integer Values

Permissions are stored as a bitfield integer. You can create permissions from an integer:

```python
# Administrator permission (value: 8)
perms = Permissions(8)

# Multiple permissions using bitwise operations
perms = Permissions((1 << 11) | (1 << 13))  # send_messages + manage_messages
```

### Using Factory Methods

The `Permissions` class provides several factory methods for common permission sets:

```python
# No permissions
perms = Permissions.none()

# All permissions
perms = Permissions.all()

# All channel permissions (excludes guild-specific ones)
perms = Permissions.all_channel()

# General permissions
perms = Permissions.general()

# Text channel permissions
perms = Permissions.text()

# Voice channel permissions
perms = Permissions.voice()

# Stage channel permissions
perms = Permissions.stage()

# Stage moderator permissions
perms = Permissions.stage_moderator()

# Membership permissions
perms = Permissions.membership()

# Advanced permissions
perms = Permissions.advanced()
```

## Modifying Permissions

### Setting Individual Permissions

You can modify permissions after creation:

```python
perms = Permissions.none()
perms.send_messages = True
perms.embed_links = True
perms.attach_files = True
```

### Updating Multiple Permissions

Use the `update()` method to set multiple permissions at once:

```python
perms = Permissions.none()
perms.update(
    send_messages=True,
    manage_messages=True,
    embed_links=True,
    attach_files=True
)
```

## Comparing Permissions

### Equality

```python
perms1 = Permissions(send_messages=True)
perms2 = Permissions(send_messages=True)
print(perms1 == perms2)  # True
```

### Subset and Superset

```python
perms1 = Permissions(send_messages=True)
perms2 = Permissions(send_messages=True, manage_messages=True)

# Check if perms1 is subset of perms2
print(perms1.is_subset(perms2))  # True
print(perms1 <= perms2)  # True

# Check if perms2 is superset of perms1
print(perms2.is_superset(perms1))  # True
print(perms2 >= perms1)  # True

# Strict comparisons
print(perms1.is_strict_subset(perms2))  # True
print(perms1 < perms2)  # True
```

## Using Permissions with OAuth2

### Basic Bot Authorization

```python
from flask_discord.types import Permissions

@app.route("/add-bot")
def add_bot():
    perms = Permissions(
        view_channel=True,
        send_messages=True,
        embed_links=True,
        attach_files=True,
        read_message_history=True,
        add_reactions=True
    )
    return discord.create_session(scope=["bot"], permissions=perms)
```

## Iterating Over Permissions

You can iterate over permissions to see which are enabled:

```python
perms = Permissions(send_messages=True, manage_messages=True)

for perm_name, value in perms:
    if value:
        print(f"{perm_name} is enabled")
```

Output:
```
send_messages is enabled
manage_messages is enabled
```

## Best Practices

<Steps>
  <Step title="Request minimum necessary permissions">
    Only request permissions your bot actually needs. This builds trust and increases installation rates.
  </Step>
  <Step title="Explain why you need permissions">
    On your authorization page, explain what each permission is used for.
  </Step>
  <Step title="Avoid administrator permission">
    The administrator permission grants complete control. Use specific permissions instead whenever possible.
  </Step>
  <Step title="Use factory methods for common sets">
    Use `Permissions.text()`, `Permissions.voice()`, etc. for standard permission sets.
  </Step>
</Steps>

<Note>
Permissions in Flask-Discord are only used during bot authorization with the `bot` scope. They determine what permissions your bot has when added to a guild.
</Note>
